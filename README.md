# Моё тестовое приложение

Возникает вопрос: как накатить миграции в БД?

Сначала в терминале нужно выполнить:

```bash
alembic init migrations
```

После этого появятся папка с миграциями и конфигурационный файл Alembic.

Дальше в `alembic.ini` нужно указать адрес базы данных, в которую будут применяться миграции.

После этого переходим в папку `migrations`, открываем `env.py` и вносим необходимые изменения в блок импорта моделей, например:

```python
from myapp import mymodel
```

Когда модели изменяются, нужно создавать новую миграцию командой:

```bash
alembic revision --autogenerate -m "comment"
```

После этого будет создан новый файл миграции.

Чтобы применить миграции к базе данных, нужно выполнить:

```bash
alembic upgrade heads
```

Чтобы во время тестов миграции создавались и применялись корректно:

1. Сначала попробуй запустить тесты обычным образом. При первом запуске они могут упасть.
2. Если после падения в папке `tests` появились alembic-файлы, нужно прописать туда данные, связанные с миграциями.
3. Если файлы не появились, зайди из консоли в папку `tests` и вызови команды для миграций вручную, чтобы эти файлы создались.
